\doxysubsection{Detailed Description}
Abstract base class representing a core module within a ship system. This class defines core functionalities, properties, and events related to a module\textquotesingle{}s state, health, and management. 


\begin{DoxyItemize}
\item {\bfseries{State Management\+:}}
\begin{DoxyItemize}
\item Tracks the module\textquotesingle{}s overall state (e.\+g., Starting Up, Running, Shutting Down) through an interface (ICore\+Module.\+Core\+Module\+State). This state can change and trigger the {\ttfamily On\+Core\+Module\+State\+Change} event.
\item Tracks the module\textquotesingle{}s operational state (e.\+g., Active, Idle, Offline) through an interface (ICore\+Module.\+Module\+Operational\+State). This state can represent operational readiness and trigger the {\ttfamily On\+Module\+Operational\+State\+Change} event. 
\end{DoxyItemize}
\end{DoxyItemize}


\begin{DoxyItemize}
\item {\bfseries{\doxylink{class_health}{Health} Management\+:}}
\begin{DoxyItemize}
\item Provides an {\ttfamily Internal\+Module\+Health} component for handling health changes (damage and healing).
\item Exposes methods for taking damage ({\ttfamily Take\+Damage}) and receiving healing ({\ttfamily Heal\+Object}).
\item Triggers {\ttfamily On\+Damage\+Event} and {\ttfamily On\+Heal\+Event} (with {\ttfamily \doxylink{interface_i_core_module_a3846425da4ecc978dd2a2da4141e1e53a605669cab962bf944d99ce89cf9e58d9}{Module\+State\+Change\+Type.\+Health}}) upon health changes. 
\end{DoxyItemize}
\end{DoxyItemize}


\begin{DoxyItemize}
\item {\bfseries{Manager Registration\+:}}
\begin{DoxyItemize}
\item Allows registering with a {\ttfamily \doxylink{class_core_ship_module_manager}{Core\+Ship\+Module\+Manager}} using {\ttfamily Register\+Core\+Module\+Manager}. This establishes a link between the module and the manager for potential coordination.
\item Provides {\ttfamily Attempt\+To\+Link\+Manager} for internal use during initialization to link with a provided manager.
\item Offers virtual methods for {\ttfamily Deregister\+Core\+System\+Manager} to break the link with the manager if needed. 
\end{DoxyItemize}
\end{DoxyItemize}


\begin{DoxyItemize}
\item {\bfseries{Initialization and Virtual Methods\+:}}
\begin{DoxyItemize}
\item Defines an {\ttfamily Initialize\+Module} method for potential module-\/specific initialization (currently calls {\ttfamily m\+\_\+internal\+Module\+Health.\+Initialize\+Health}).
\item Provides abstract methods for core functionalities\+:
\begin{DoxyItemize}
\item {\ttfamily Release\+Resources}\+: Needs implementation for releasing resources when a module is no longer needed.
\item {\ttfamily Shut\+Down}\+: Requires implementation for handling module shutdown procedures.
\item {\ttfamily Start\+Up}\+: Requires implementation for handling module startup procedures.
\item {\ttfamily Reboot}\+: Requires implementation for handling module reboot procedures. 
\end{DoxyItemize}
\end{DoxyItemize}
\end{DoxyItemize}


\begin{DoxyItemize}
\item {\bfseries{Events\+:}}
\begin{DoxyItemize}
\item {\ttfamily On\+Core\+Module\+State\+Change}\+: Triggered when the module\textquotesingle{}s overall state changes.
\item {\ttfamily On\+Module\+Operational\+State\+Change}\+: Triggered when the module\textquotesingle{}s operational state changes.
\item {\ttfamily On\+Damage\+Event}\+: Triggered when the module takes damage.
\item {\ttfamily On\+Heal\+Event}\+: Triggered when the module receives healing.
\item {\ttfamily On\+Death\+Event}\+: Potentially intended to be triggered when the module is destroyed due to health depletion . 
\end{DoxyItemize}
\end{DoxyItemize}

{\bfseries{Inheritance\+:}} This class is marked as abstract, meaning it cannot be directly instantiated. Concrete core module classes should inherit from {\ttfamily \doxylink{class_b_c___core_module}{BC\+\_\+\+Core\+Module}} and implement the abstract methods to provide specific functionalities.
